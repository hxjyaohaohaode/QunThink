# AI人设自定义与用户画像系统 Spec

## Why
当前AI人设是硬编码的，回复风格模板化严重，不像真人聊天。用户无法自定义AI的性格和回复方式，也无法让AI了解自己的个人信息，导致AI无法提供个性化服务。需要让AI像真人一样自由交流，并支持用户自定义AI人设和个人画像。

## What Changes
- **BREAKING**: 重构AI人设系统，从硬编码改为可动态配置，用户可在侧边栏自定义每个AI的名称、风格、回复方式等
- 在侧边栏"在线成员"区域添加AI人设设置功能（点击AI成员可编辑人设）
- 在侧边栏添加用户个人画像设置模块（身高、体重、年龄、性别、爱好、性格、学历、目标、自我介绍等）
- 重写系统提示词，注入用户画像信息和自定义人设，使AI回复更加个性化、真人化
- 大幅增加上下文窗口和对话摘要容量，确保AI记忆完整
- 优化AI回复去模板化，增加口语化、情绪化表达

## Impact
- Affected specs: AI人设配置、系统提示词构建、侧边栏UI、用户数据模型
- Affected code:
  - `backend/src/config/personas.js` - 人设从硬编码改为可配置
  - `backend/src/services/ai/index.js` - 系统提示词重构，上下文窗口扩大
  - `backend/src/services/scheduler/index.js` - 调度器消息读取量增加
  - `backend/src/models/db.js` - 新增用户画像数据结构
  - `backend/src/routes/` - 新增用户画像和AI人设API
  - `frontend/src/components/Layout/Sidebar.tsx` - 侧边栏增加人设设置和用户画像
  - `frontend/src/stores/` - 新增用户画像和AI人设状态管理
  - `frontend/src/services/api.ts` - 新增API调用

## ADDED Requirements

### Requirement: AI人设自定义设置
系统应允许用户在侧边栏中自定义每个AI的人设参数。

#### Scenario: 编辑AI人设
- **WHEN** 用户点击侧边栏中的AI成员名称
- **THEN** 弹出该AI的人设编辑面板
- **AND** 可编辑字段包括：昵称、风格标签、回复方式、性格描述、典型用语（每行一个）
- **AND** 修改后实时保存到后端，下次AI回复时使用新人设

#### Scenario: 重置AI人设
- **WHEN** 用户在人设编辑面板点击"恢复默认"
- **THEN** AI人设恢复为系统默认值
- **AND** 前端显示更新为默认人设

### Requirement: 用户个人画像设置
系统应允许用户设置个人画像信息，AI在回复时参考这些信息。

#### Scenario: 设置用户画像
- **WHEN** 用户在侧边栏点击"我的信息"区域
- **THEN** 弹出用户画像编辑面板
- **AND** 可编辑字段包括：昵称、性别、年龄、身高、体重、职业、学历、爱好（多选标签）、性格（多选标签）、目标、自我介绍
- **AND** 保存后AI在后续回复中参考用户画像

#### Scenario: AI感知用户画像
- **WHEN** 用户设置了画像信息并发送消息
- **THEN** AI的系统提示词中包含用户画像摘要
- **AND** AI回复时体现对用户个人情况的理解和个性化建议

### Requirement: AI回复去模板化
系统应确保AI回复自然、口语化，不使用模板化表达。

#### Scenario: 系统提示词去模板化
- **WHEN** 构建AI系统提示词时
- **THEN** 提示词中明确禁止使用模板化表达（如"首先...其次...最后..."、"综上所述"等）
- **AND** 鼓励AI使用口语化、有情绪、有态度的表达方式
- **AND** 提示词中注入用户自定义的AI人设信息

#### Scenario: AI自由表达
- **WHEN** AI生成回复时
- **THEN** AI可以自由选择回复方式：赞同、反驳、追问、调侃、感叹等
- **AND** 不再强制按固定格式回复
- **AND** 回复长度灵活，短到一句话、长到几段话均可

### Requirement: 超大上下文记忆
系统应支持超大上下文窗口，确保AI记住完整对话历史。

#### Scenario: 上下文窗口扩展
- **WHEN** AI处理用户消息时
- **THEN** 上下文窗口支持最多1000条消息
- **AND** 系统提示词中的对话摘要支持最近200条消息
- **AND** 调度器读取消息量增加到1000条

#### Scenario: 智能上下文截断
- **WHEN** 消息数量超过API token限制时
- **THEN** 系统智能截断较早的消息，保留最近的消息
- **AND** 对被截断的消息生成简要摘要，附加在系统提示词中

## MODIFIED Requirements

### Requirement: AI人设配置
原有人设硬编码在 `personas.js` 中，现改为从数据库读取，支持用户自定义覆盖。

**原逻辑**：
- 人设从 `personas.js` 静态导入，不可修改

**新逻辑**：
- 默认人设仍从 `personas.js` 读取
- 用户自定义人设存储在数据库中，优先级高于默认值
- 系统提示词使用合并后的人设信息

### Requirement: 系统提示词构建
原有系统提示词使用固定模板和少量上下文，现改为动态构建，包含用户画像和完整对话历史。

**原逻辑**：
- 固定模板 + 最近30条对话摘要

**新逻辑**：
- 动态人设注入 + 用户画像摘要 + 最近200条对话摘要 + 智能截断

## REMOVED Requirements
无移除的需求。
