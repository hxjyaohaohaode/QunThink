# 群聊自然交互与未读消息体验优化 — 验收检查清单

## AI自然发言调度验证

- [x] AI发言不再有明显的顺序感（不再出现A→B→C→D的固定模式）
- [x] 多个AI的"正在输入"状态独立出现，不是同时出现
- [x] 部分AI会选择沉默不发言（不是所有AI都每次必发言）
- [x] AI发言间隔自然分布（1~8秒范围，有快有慢）
- [x] 被@提及的AI仍然优先发言
- [x] 辩论模式下的发言逻辑不受影响

## AI禁止复制发言验证

- [x] AI回复中不出现与前面发言高度相似的段落
- [x] AI引用他人观点时使用自己的语言重新表达
- [x] system prompt中包含明确的"禁止复制"规则
- [x] normalizeResponse中有去重检测逻辑

## AI智能评论验证

- [x] AI评论内容与目标消息主题相关
- [x] AI评论不再是固定的"同意这个观点！"等硬编码话术
- [x] AI评论通过WebSocket实时推送到前端
- [x] AI评论API失败时降级为基于关键词的模板评论
- [x] 评论触发时机自然，不会在消息刚出现时就评论

## 进入群聊自动滚动验证

- [x] 用户切换到某个群聊时，消息列表自动滚动到最底部
- [x] 首次打开应用进入默认群聊时，自动滚动到最新消息
- [x] 滚动完成后isAtBottom状态正确设为true
- [x] 滚动不会出现闪烁或抖动

## 微信风格未读消息提示验证

- [x] 用户浏览历史消息时，新消息到达不自动跳转
- [x] 右下角显示未读消息数量徽章
- [x] 徽章样式参考微信风格（圆角胶囊形、绿色主题色、向下箭头）
- [x] 点击徽章跳转到第一条未读消息位置（非底部）
- [x] 第一条未读消息有短暂高亮动画效果
- [x] 跳转后如果仍有未读消息，徽章显示剩余数量
- [x] 手动滚动到底部附近时，徽章自动消失
- [x] 未读计数准确，不会出现负数或错误计数

## 点赞/评论视觉反馈验证

- [x] AI点赞后，对应消息的点赞数实时增加
- [x] AI评论后，对应消息下方实时显示新评论
- [x] 新点赞和新评论有入场动画效果
- [x] 点赞者标识正确显示（AI名称而非ai_xxx格式）
